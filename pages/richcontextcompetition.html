Rich Context Competition

COMPETITION GOAL

The goal of this competition is to develop automated methods that will scour social science publication corpora in order to confidently identify datasets that are used in research publications. Through partnering with the Interuniversity Consortium for Political and Social Research (ICPSR) and the Deutsche Bundesbank, we are making available manually matched publications which reference datasets commonly used in social science and economic research. We are also partnering with JSTOR, to provide a large corpus of publication text.

APPROACH AND BACKGROUND

Our long range goal is to change the empirical foundation of social science, statistical, and public agencies in the United States and transform our understanding of how our society works. We do this by building a computational research platform – the NYU Administrative Data Research Facility in collaboration with ICPSR and Project Jupyter - that enables data science and machine learning to be used by organizations working with sensitive and confidential microdata.

Our approach is to build a set of tools for this platform that enable collaborative knowledge creation and discovery with confidential microdata. We propose to accomplish this through rich contextual metadata about the data sets: user created comments/annotations, Jupyter notebooks with analysis, and machine-learned relationships among the data sets and publications/researchers who have used the data set.

There are three operational steps. The first is to use text analysis and machine learning to discover relationships between data sets, researchers, publications and research method. The second is to present the rich context of those relationships to users of the computational platform as they work. The third is the use of human centered design to create user experiences that incentivize those users to a) create and share additional rich context metadata (comments, annotations, analysis code) and b) contribute new tacit knowledge that was hitherto not routinely shared about a dataset or related work.

The results will be threefold. First, researchers, analysts, and students will vastly speed up and enhance their own pre-analysis work (e.g., research design, literature review) because when they begin to work with sensitive data, they will be able to see what other research has been done, by whom, and with what results. Second, social science research will be transformed from a largely solitary activity to a collaborative one – where researchers are rewarded and credentialed for contributions of code, analysis and data annotations. Third, the ability to measure the value of data through new “alt metrics” of data impact and use will encourage the wider replication of published empirical work and boost the capacity of currently marginalized data collectors and providers such as statistical and public agencies to make their data more accessible.

HOW TO PARTICIPATE

We welcome the participation of US and international researchers who can bring creative new technical approaches to this research problem. We invite you to review the workshop and technical parameters below, explore the publications data, and send a brief note of your “intent to participate” to rich_context@nyu.edu by {MONTH-DAY-YEAR}. All reasonable submissions will be accepted for participation.

In your intent to participate please provide the following details:

  • The outline of an algorithmic approach around which you will develop appropriate feature engineering steps and a means by which to validate and test your model.
  • Describe any external datasets or non-traditional computing environments that you wish to incorporate in your proposed solution.
  • Give a rough description of your development platform, including the operating system, CPU specifications, the amount of available memory, and GPU specifications if applicable.*
  • List any non-free software that you will use.

*We understand that the exact specifications of your platform may change

REMUNERATION

One $25,000 stipend will be provided to the research team with the algorithm that scores highest upon evaluation.

  • That stipend will support the team’s work on their algorithm.
  • The stipend will additionally support the research team’s technical guidance as Rich Context staff integrate their work into the broader project.

The Rich Context project will provide a stipend to cover travel expenses for one representative from up to 15 participating teams. If more than 15 teams submit eligible algorithms, the evaluation results will be used to identify the 15 teams that will be represented at the workshop.

PROGRAM REQUIREMENTS

Participants will write a program that reads in files containing the supplied processed text data and produce an output file giving predictions for which datasets are identified in the text.

Input files

The input files consist of {####} plain text publications along with metadata about the publications and the datasets referenced in the curated corpus. Participants are free to use any portion of this data for their algorithms as they see fit. The metadata are provided in JSON format and can be easily linked to the text input files.

Output file format

The output file should be a tab-delimited file with two columns and no header. The first column should be the document id. The second column should be an integer ID that identifies the cited dataset.

The algorithm should not run for more than 5 days when processing all publications text and dataset metadata.

The implementation should be runnable on hardware equivalent to a single Amazon Web Services (AWS) instance. The panel will review any requests for software or hardware updates that might be required to accommodate the incorporation of a novel algorithm into the proposed infrastructure. These requests must be submitted in your letter of intent to participate.

External data

  • Proprietary datasets cannot be included in any algorithm.
  • The panel will review any requests to incorporate additional non-proprietary data into a submitted algorithm. Please specify any additional data you intend to use in your letter of intent to participate.

EVALUATION

Algorithms will be evaluated for accuracy, run-time, usability, and novelty. The evaluation will take place in two phases.

First Phase

This is an initial round of self-testing where participants will infer datasets from the corpus. They may train their algorithms using any part of the provided data, as well as any additional data sets that have been submitted to the workshop organizers.

They will submit a model program including:

  • Trained model, ready to be run.
  • Documentation on how to install, train, configure, and run the model program.
  • All source code and data used to train and test the model
  • Written high-level summary of what the program does, including intermediate steps in the process and external data sources used.
  • Written high-level summary of novelty of additional metadata.

During this round, participants will be evaluated by the expert team using the following criteria:

  • Recall Rate defined as Recall = (# of true positives) / ((# of true positives) + (# of false negatives))
  • Precision Rate defined as Precision = (# of true positives) / ((# of true positives) + (# of false positives))
  • Self-reported algorithm run-time
  • Quality of implementation, including:
    • Quality of code (Scored by expert team)
    • Replicability and generalizability (Scored by expert team)
    • Documentation (Scored by expert team)

Second Phase

Up to six competition participants will be invited to a second evaluation phase, where they will be asked to run their disambiguation algorithm on a set of 100,000 documents in a server environment that we provide.

During this phase, participants will be evaluated on:

  • Algorithm generalizability, which we will assess by computing the recall rate and precision rate for different sets of training and evaluation data.
  • Self-reported algorithm run-time.
  • Novelty of additional metadata for each publication, including:
    • Methods used (Scored by expert team)
    • Topics identified (Scored by expert team)
    • Research findings (Scored by expert team)
  • Quality of implementation, including:
    • Quality of code
    • Replicability and generalizability
    • Documentation
